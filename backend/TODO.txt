## 🧩 **Модуль 1: CONNECTIONS (2 часа) — САМОЕ ВАЖНОЕ**

### 1.1 Модель Connection
```python
# Модель для хранения связей между любыми сущностями
```
**Поля:** id, from_entity_id, to_entity_id, type, status, created_at, created_by, metadata (JSON для доп. данных)

### 1.2 Сервис ConnectionService
**Методы:**
- `create(from_id, to_id, type, user_id, metadata=None)` — создать связь
- `get_by_entity(entity_id)` — все связи сущности (входящие и исходящие)
- `get_between(entity1_id, entity2_id)` — есть ли связь между двумя
- `update_status(connection_id, status, user_id)` — изменить статус
- `delete(connection_id, user_id)` — удалить связь

### 1.3 Эндпоинты
- `POST /connections` — создать (авторизация)
- `GET /connections/entity/{id}` — все связи сущности
- `GET /connections/entity/{id}/incoming` — только входящие (кто ищет меня)
- `GET /connections/entity/{id}/outgoing` — только исходящие (кого ищу я)
- `PUT /connections/{id}/status` — обновить статус (pending → active → completed)
- `DELETE /connections/{id}` — удалить

### 1.4 Типы связей (продумать)
- `collaboration` — уже сотрудничают
- `seeks` — ищет партнёра
- `offers` — предлагает услуги
- `past` — работали раньше

## 📊 **Модуль 4: STATS (1 час) — про коллаборации**

### 4.1 Эндпоинт /stats/collaborations
**Данные:**
- total_connections — всего связей
- by_type — разбивка по типам (collaboration, seeks, offers)
- by_status — pending, active, completed
- top_collaborators — сущности с наибольшим числом связей

### 4.2 Эндпоинт /stats/network
**Данные для графа:**
- nodes: все сущности (id, name, type)
- edges: все активные связи (from, to, type)

### 4.3 Эндпоинт /stats/entity/{id}/network
**Данные:** Граф связей конкретной сущности (кто с кем связан через неё).

### 4.4 Эндпоинт /stats/tags/correlation
**Что делает:** Показывает, какие теги чаще всего встречаются вместе (для рекомендаций).

## 💡 **КЛЮЧЕВЫЕ ПОНЯТИЯ ДЛЯ ПОНИМАНИЯ**

| Термин | Что это |
|--------|---------|
| **Узел (node)** | Сущность (команда, проект, компания) |
| **Ребро (edge)** | Связь между сущностями |
| **Тип связи** | Вид коллаборации (сотрудничают, ищут, предлагают) |
| **Статус** | pending (запрос), active (сотрудничают), completed (завершено) |
| **Граф** | Визуализация сети связей |
